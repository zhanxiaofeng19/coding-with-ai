# Prompt 模式：性能优化

## 使用场景
用户需要优化接口性能、降低延迟、提高吞吐量时使用此模式。

## 必须提供的上下文
1. 性能问题描述（哪个接口慢、慢多少）
2. 当前指标（QPS、P99、错误率）
3. 目标指标
4. 约束条件（不能改的部分）

## Agent 工作流程

### Step 1: 性能分析
- 分析慢在哪里（DB / 缓存 / 远程调用 / 计算）
- 识别瓶颈点
- 量化每个环节耗时

### Step 2: 优化方案
按优先级排列优化手段：

#### 第一优先级：减少调用
- 合并重复查询
- 移除不必要的远程调用
- 批量替代逐条操作

#### 第二优先级：增加缓存
- 本地缓存（Caffeine）
- 分布式缓存（Redis）
- 多级缓存

#### 第三优先级：异步化
- 非核心操作异步化（MQ / 线程池）
- 并行调用独立的依赖

#### 第四优先级：数据库优化
- SQL 优化（索引、查询改写）
- 读写分离
- 分表

#### 第五优先级：架构优化
- 预计算
- 数据冗余
- CQRS

### Step 3: 实施
- 每个优化独立提交
- 每步验证性能提升
- 记录优化前后的指标对比

### Step 4: 验证
- 压测验证优化效果
- 确认无功能回归
- 更新 SLO 文档
