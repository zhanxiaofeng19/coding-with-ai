# Prompt 模式：故障排查

## 使用场景
线上出现故障、异常告警时使用此模式。

## 必须提供的上下文
1. 故障现象
2. 影响范围
3. 开始时间
4. 最近是否有发布/变更
5. 相关告警信息

## Agent 工作流程

### Step 1: 快速评估
- 确认影响范围（哪些服务、哪些用户）
- 判断严重级别
- 是否需要立即止血

### Step 2: 止血优先
如果需要立即止血，优先建议：
- 回滚最近的发布
- 关闭特性开关
- 降级受影响的功能
- 扩容
- 限流

### Step 3: 根因定位
按以下顺序排查：
1. **最近变更**：检查最近的发布、配置变更、DDL 变更
2. **监控指标**：QPS、延迟、错误率、资源使用率
3. **日志分析**：错误日志、异常堆栈
4. **链路追踪**：慢请求链路、异常链路
5. **依赖状态**：DB、Redis、MQ、外部服务

### Step 4: 修复方案
- 根据根因制定修复方案
- 评估修复的风险
- 必要时先在预发验证

### Step 5: 修复验证
- 确认指标恢复正常
- 确认用户影响消除
- 检查数据是否需要修复

### Step 6: 输出复盘报告
使用 `specs/templates/incident-runbook-template.md` 记录：
- 时间线
- 根因
- 影响范围
- 止血和修复动作
- 预防措施
- 改进项

## 常见故障模式
| 现象 | 可能原因 | 排查方向 |
|------|---------|---------|
| 接口 5xx 飙升 | 代码 Bug / 依赖挂了 | 最近发布 + 依赖状态 |
| 接口延迟飙升 | 慢查询 / 缓存击穿 / GC | DB 慢日志 + 缓存命中率 + JVM |
| 消息积压 | 消费异常 / 消费变慢 | 消费日志 + 重试次数 |
| 连接池耗尽 | 慢查询 / 连接泄漏 | 连接池监控 + 慢日志 |
| OOM | 内存泄漏 / 大对象 | Heap Dump + GC 日志 |
