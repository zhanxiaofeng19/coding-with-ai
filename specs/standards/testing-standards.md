# 测试规范

## 单元测试
- 核心逻辑必须有单测
- 覆盖：正常流 + ≥2 异常流
- 命名：should_动作_when_条件
- Mock 隔离外部依赖
- 不依赖外部服务

## 覆盖率
- Service 核心方法 ≥ 80%
- 工具类 ≥ 90%
- 整体 ≥ 60%

## 集成测试
- Testcontainers / H2
- Mock 外部服务
- 完整请求→响应链路
- 验证数据库状态

## 必覆盖场景
- 空值 / null
- 边界值（0、负数、最大值）
- 空集合
- 并发修改
- 异常路径
- 幂等重试

## 压测
- 核心接口上线前压测
- 目标：日常 QPS × 3~5
- 独立环境
- 报告含 QPS、P99、错误率、资源使用率

## 质量门禁
- 编译通过
- 单测通过
- 覆盖率达标
- 静态扫描无 BLOCKER/CRITICAL
