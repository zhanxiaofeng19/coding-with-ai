# 灰度发布规范

## 灰度策略

### 流量百分比灰度
- 1% → 5% → 20% → 50% → 100%
- 每阶段观察 ≥ 15 分钟
- 任何阶段指标异常立即回滚

### 维度灰度
- 用户 ID 尾号灰度
- 城市维度灰度
- 渠道维度灰度
- 可组合使用

## 观察指标

### 核心指标（必须观察）
- 接口错误率
- 接口 P99 延迟
- 核心业务转化率
- 异常日志量

### 资源指标
- CPU 使用率
- 内存使用率
- 线程池使用率
- 连接池使用率

## 回滚条件

满足任一条件立即回滚：
- 错误率 > 基线 5 倍
- P99 > 基线 3 倍
- 核心业务指标下跌 > 10%
- 出现未预期的异常类型

## 灰度流程

1. **准备**：确认灰度计划、回滚方案、观察指标
2. **预发验证**：预发环境全量验证
3. **小流量**：1% 流量灰度，观察 15min
4. **扩大**：逐步扩大，每阶段观察 15min
5. **全量**：全量发布
6. **观察**：全量后观察 ≥ 30min
7. **清理**：清理灰度配置和特性开关

## 灰度规则
- 灰度期间保持新旧版本共存兼容
- 灰度期间数据库双写（如需要）
- 灰度用户的操作日志单独标记
- 灰度异常时优先止血，再分析原因
