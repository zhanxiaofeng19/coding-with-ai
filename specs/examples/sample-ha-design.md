# 示例：高可用设计 — 订单创建接口

## 依赖分析

| 依赖 | 类型 | 超时 | 是否核心 | 降级方案 |
|------|------|------|---------|---------|
| MySQL | 数据库 | 500ms | 核心 | 无法降级，返回系统繁忙 |
| Redis | 缓存 | 100ms | 非核心 | 跳过缓存直接查 DB |
| 库存服务 | RPC | 1s | 核心 | 熔断后返回"库存确认中" |
| 优惠券服务 | RPC | 1s | 非核心 | 熔断后跳过优惠 |
| RocketMQ | 消息 | 3s | 非核心 | 写本地消息表补偿 |

## 超时链路

```
客户端(5s) → 网关(3s) → 订单服务(2s) → 库存服务(1s)
                                      → 优惠券服务(1s)
                                      → MySQL(500ms)
                                      → Redis(100ms)
```

## 熔断配置

| 依赖 | 错误率阈值 | 窗口期 | 半开间隔 | 降级行为 |
|------|-----------|--------|---------|---------|
| 库存服务 | 50% | 10s | 5s | 返回"库存确认中"，异步重试 |
| 优惠券服务 | 50% | 10s | 5s | 跳过优惠计算 |

## 限流配置

| 接口 | 限流维度 | 阈值 | 算法 |
|------|---------|------|------|
| 创建订单 | 用户 | 10次/秒 | 令牌桶 |
| 创建订单 | 全局 | 2000次/秒 | 滑动窗口 |

## 容量评估

| 指标 | 日常 | 峰值 | 当前容量 | 是否充足 |
|------|------|------|---------|---------|
| QPS | 500 | 2500 | 5000 | 是 |
| MySQL 连接 | 50 | 150 | 200 | 是 |
| Redis 内存 | 2GB | 4GB | 16GB | 是 |

## 回滚方案

| 场景 | 回滚方式 |
|------|---------|
| 代码 Bug | K8s 回滚到上一版本 |
| 数据异常 | 根据 orderId 范围修复 |
| 配置错误 | 配置中心回滚 |
