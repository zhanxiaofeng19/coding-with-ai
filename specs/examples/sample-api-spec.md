# 示例：API 设计文档 — 创建订单

## 修改历史

| 日期 | 修改人 | 修改内容 |
|------|--------|---------|
| 2026-04-02 | 张三 | 初始版本 |

## 1. 接口概述

| 项目 | 内容 |
|------|------|
| 接口名称 | 创建订单 |
| 请求方式 | POST |
| 请求路径 | /api/v1/orders |
| 所属模块 | 订单 |
| 是否幂等 | 是 |
| 幂等机制 | 基于 requestId 去重 |
| 鉴权方式 | Bearer Token |
| 限流策略 | 10 次/秒/用户 |

## 2. 请求参数

### Header

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Authorization | String | 是 | Bearer Token |
| X-Request-Id | String | 是 | 幂等键（客户端生成的 UUID） |

### Body（JSON）

```json
{
  "productId": 1001,
  "quantity": 2,
  "addressId": 5001,
  "couponId": null,
  "remark": "请尽快发货"
}
```

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|---------|------|
| productId | Long | 是 | > 0 | 商品 ID |
| quantity | Integer | 是 | 1~99 | 购买数量 |
| addressId | Long | 是 | > 0 | 收货地址 ID |
| couponId | Long | 否 | - | 优惠券 ID |
| remark | String | 否 | ≤ 200 字符 | 订单备注 |

## 3. 响应

### 成功响应

```json
{
  "code": "SUCCESS",
  "message": "操作成功",
  "data": {
    "orderId": 1234567890
  },
  "traceId": "trace_abc123"
}
```

### 错误响应

| 错误码 | HTTP 状态码 | 说明 |
|--------|-----------|------|
| 20_BIZ_001 | 200 | 订单已关闭 |
| 20_BIZ_002 | 200 | 库存不足 |
| 20_BIZ_003 | 200 | 优惠券不可用 |
| 00_PARAM_001 | 400 | 参数不能为空 |
| 00_AUTH_001 | 401 | Token 过期 |
| 00_LIMIT_001 | 429 | 请求过于频繁 |

| 指标 | 目标值 |
|------|--------|
