## 检查清单

### 功能正确性

- [ ] 实现是否符合设计文档
- [ ] 边界条件是否处理
- [ ] 异常路径是否覆盖
- [ ] 幂等处理是否完整

### 代码质量

- [ ] 命名是否符合团队约定
- [ ] 分层是否正确（Controller/Service/Manager/Mapper）
- [ ] 是否有重复代码可抽取
- [ ] 方法复杂度是否过高
- [ ] 注释是否必要且准确

### 性能

- [ ] 是否有 N+1 查询
- [ ] 是否有不必要的远程调用
- [ ] 事务范围是否最小化
- [ ] 大集合是否分批处理

### 并发安全

- [ ] 共享状态是否有保护
- [ ] 锁的粒度和范围是否合理
- [ ] 是否有死锁风险

### 数据安全

- [ ] SQL 是否参数化
- [ ] 用户输入是否校验
- [ ] 敏感数据是否脱敏
- [ ] 权限检查是否完整

### 可观测性

- [ ] 关键日志是否完整
- [ ] traceId 是否贯穿
- [ ] 监控指标是否覆盖

### 兼容性

- [ ] 接口是否向前兼容
- [ ] 数据格式是否兼容
- [ ] 配置变更是否安全

### 测试

- [ ] 单元测试是否补充
- [ ] 核心路径是否覆盖
- [ ] 异常路径是否覆盖

## Review 结论

- [ ] 通过
- [ ] 需修改后通过
- [ ] 需要重新 Review
