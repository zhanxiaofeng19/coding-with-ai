## 检查清单

### 1. 单点风险

- [ ] 是否存在单点依赖（只有一个实例/节点）
- [ ] 单点挂掉后的影响范围
- [ ] 是否有主备/多副本部署

### 2. 超时与重试

- [ ] 所有远程调用是否设置了超时
- [ ] 超时时间是否合理（< 上游超时的 80%）
- [ ] 重试操作是否幂等
- [ ] 重试次数是否有上限
- [ ] 是否有重试风暴风险

### 3. 熔断与降级

- [ ] 外部依赖是否配置了熔断
- [ ] 熔断后的降级行为是否明确
- [ ] 降级是否可配置化（开关控制）
- [ ] 降级后用户体验是否可接受

### 4. 限流

- [ ] 核心写接口是否有限流
- [ ] 限流策略是否合理（用户/IP/接口维度）
- [ ] 被限流后的返回信息是否友好

### 5. 缓存风险

- [ ] 缓存穿透防护
- [ ] 缓存击穿防护
- [ ] 缓存雪崩防护（TTL 抖动）
- [ ] 缓存一致性方案
- [ ] 热点 key 防护

### 6. 队列风险

- [ ] 消息积压应急方案
- [ ] 消费者幂等处理
- [ ] 死信队列配置和告警
- [ ] 消息丢失防护

### 7. 数据库风险

- [ ] 是否有慢查询风险
- [ ] 是否有锁竞争风险
- [ ] 大表 DDL 方案
- [ ] 读写分离是否正确使用

### 8. 容量评估

- [ ] QPS 预估（日常 + 峰值）
- [ ] 数据库容量是否充足
- [ ] 缓存容量是否充足
- [ ] 消息队列容量是否充足
- [ ] 是否需要预扩容

### 9. 可观测性

- [ ] 核心指标是否有监控
- [ ] 告警规则是否配置
- [ ] 日志是否完整（入口/异常/关键分支）
- [ ] traceId 是否贯穿

### 10. 回滚方案

- [ ] 代码回滚是否可行
- [ ] 数据回滚方案
- [ ] 配置回滚方案
- [ ] 回滚后是否需要数据修复
